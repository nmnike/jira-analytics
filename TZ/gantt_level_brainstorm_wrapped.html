A — Портфельный Gantt

Каждая строка — одна инициатива. Видна картина квартала целиком: что когда, где пересечения, где перегрузка команды. Под диаграммой — тепловая карта загрузки по ролям.

  • 10–30 строк на квартал
  • Тепловая карта загрузки по ролям
  • Быстро строится — данные уже есть
  • Целевая аудитория: PM, руководство

B — Двухуровневый Gantt

Инициативы раскрываются до уровня эпиков. Видны зависимости между эпиками, можно понять, что блокирует что. Баланс между обзором и детализацией.

  • 50–150 строк при полном раскрытии
  • Стрелки зависимостей между эпиками
  • Данные из Jira уже синхронизированы
  • Целевая аудитория: PM + тимлиды

C — Полный task-level Gantt

Инициатива → Эпик → Story → Подзадача. Критический путь рассчитывается по всем задачам. Зависимости из Jira-поля "blocks". Уровень детализации MS Project.

  • 500–2000 строк при полном раскрытии
  • Автоматический расчёт критического пути
  • Сложно строить, сложно читать
  • Целевая аудитория: тимлиды, PMO
