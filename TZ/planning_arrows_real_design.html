Сравнительная таблица

Вариант Плюсы Минусы
1. DHTMLX-style
Толстые, контрастные
✓ Максимальная читаемость зависимостей
✓ Явное разделение фаза vs ресурс по цвету
✓ Тень даёт глубину, линии не теряются на барах
✗ Может загромождать плотный граф
✗ Визуально «кричит» — отвлекает от содержания баров
✗ Стиль DHTMLX читается как «сторонняя библиотека»
2. Halo
Двойной контур
✓ Элегантно и профессионально
✓ Линии читаются на любом фоне благодаря подложке
✓ Тонкий верхний слой не загромождает чарт
✗ Требует двух SVG-путей на каждую линию (+нагрузка)
✗ На очень плотных чартах подложки сливаются
✗ Маркеры стрелок чуть сложнее в реализации
3. Анимированный поток
Движущийся ресурс
✓ Мгновенно понятно направление передачи ресурса
✓ Живая анимация привлекает взгляд только к нужным связям
✓ Wow-эффект при демонстрации PM и заказчику
✗ Движение отвлекает при длительной работе
✗ PDF/скриншот теряет ключевой эффект
✗ Нужна возможность выключить анимацию
4. Семантический минимализм
Один цвет, форма = смысл
✓ Не конкурирует с цветами баров
✓ Легко читается при большом числе связей
✓ Просто в реализации и поддержке
✗ Фаза vs ресурс различить только по форме, не по цвету
✗ На тёмном фоне может казаться невыразительным
✗ Теряет «дорогой» вид продукта
Рекомендация: Вариант 2 (Halo) — лучший баланс между читаемостью и визуальным качеством: линии никогда не теряются на тёмном фоне, а двойной контур добавляет профессиональный вид без излишнего шума; при желании можно добавить анимацию из варианта 3 только на эстафетные связи через тогл «Эстафета».
