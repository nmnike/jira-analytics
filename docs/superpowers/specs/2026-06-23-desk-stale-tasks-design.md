# Плитка «Залежавшиеся задачи» на рабочем столе аналитика

Дата: 2026-06-23
Статус: дизайн утверждён, путь A (правка синхронизации)

## Цель

Новая плитка на `/desk/<token>`, помогающая аналитику увидеть забытые задачи —
которые давно никто не трогал. Две колонки рядом:

- **Мои задачи** — созданы аналитиком, висят на других исполнителях.
- **Задачи мне** — назначены аналитику и стоят без движения.

«Старее» = дольше без последнего касания в Jira. Цель — находить задачи,
которые создали и забыли.

## Что считаем «последним касанием»

Поле Jira `updated` (дата последнего изменения задачи). Меняется при любом
действии: правка описания, смена статуса, ворклог, комментарий, смена полей.
Это точнее и проще, чем собирать максимум из косвенных следов
(`status_changed_at` / ворклоги / комментарии), которые мы храним сейчас.

## Изменения данных (путь A)

Сейчас в задаче **не хранятся**: автор, исполнитель как личность, дата
последнего изменения Jira. Храним только текстовое имя исполнителя
(`assignee_display_name`) и косвенные метки.

Добавляем в модель `Issue` (Alembic batch-миграция) четыре поля:

| Поле | Тип | Источник Jira | Назначение |
|---|---|---|---|
| `jira_updated_at` | DateTime (index) | `fields.updated` | дата последнего касания, главная сортировка |
| `reporter_account_id` | String(128) | `fields.creator.accountId` | автор задачи (личность) |
| `reporter_display_name` | String(255) | `fields.creator.displayName` | автор для показа |
| `assignee_account_id` | String(128) | `fields.assignee.accountId` | исполнитель (личность), может быть NULL |

`assignee_display_name` уже есть — оставляем.

Сопоставление с сотрудником: сравниваем `reporter_account_id` /
`assignee_account_id` с `Employee.jira_account_id` аналитика рабочего стола.

### Backfill

Поля заполняются при синхронизации. Чтобы заполнить уже накопленные ~115k
задач — одна полная перезагрузка задач (существующая операция full sync).
До перезагрузки плитка покажет пусто / частично — это нормально.

## Отбор задач (общий фильтр обеих колонок)

1. проект задачи — `PMD` или `OS` (по ключу проекта);
2. не завершена: `status_category != 'done'`;
3. листовая: нет подзадач (`children` пуст).

> Примечание: PRJ / ITL / AD исключаются автоматически фильтром по проекту.
> Условие «листовая» дополнительно убирает родительские задачи внутри PMD/OS.

## Логика колонок

**Мои задачи** (создал, висит на других):
- `reporter_account_id == <analyst.jira_account_id>`
- `assignee_account_id != <analyst.jira_account_id>` (включая NULL — не назначена)
- сортировка по `jira_updated_at` по возрастанию (самые давние сверху), ТОП-10
- в строке второй человек = исполнитель (`assignee_display_name`, либо «не назначен»)

**Задачи мне** (назначены, стоят):
- `assignee_account_id == <analyst.jira_account_id>`
- сортировка по `jira_updated_at` по возрастанию, ТОП-10
- в строке второй человек = автор (`reporter_display_name`)

## Строка списка

- ключ + название задачи — ссылка в Jira (`<base_url>/browse/<key>`);
- дней без изменений (от `jira_updated_at` до сегодня) — крупно, главная метрика;
- статус (бейдж, цвет по `status_category` как в остальном UI);
- второй человек (исполнитель / автор) — аватар + имя.

## Бэкенд

Новый виджет `stale_tasks`:
- добавить ключ в `WIDGET_KEYS` и `_REGISTRY` (`app/services/work_desk_widgets.py`);
- адаптер `_adapter_stale_tasks(db, desk, year, quarter)` — два запроса (мои / мне),
  каждый по фильтру выше + сортировка + limit 10;
- джойн на `Employee` по account_id для аватарок/имён, на `Project` для ключа проекта;
- `base_url` для ссылок в Jira — как уже отдаётся другим виджетам.

Возвращаемая форма:
```
{
  "base_url": "...",
  "my_tasks":   [ { key, summary, days_idle, status, status_category, person: {name, avatar_url} }, ... ],
  "assigned":   [ { key, summary, days_idle, status, status_category, person: {name, avatar_url} }, ... ]
}
```

## Фронтенд

- новый компонент `StaleTasksWidget.tsx` (`frontend/src/components/desk/`);
- регистрация в `WIDGET_REGISTRY` (`registry.tsx`), `wide: true`;
- размещение: `fullRow('stale_tasks')` сразу после `fullRow('my_timeline')`
  в `DeskLayout` (`DeskPage.tsx`);
- две колонки рядом (на узком экране складываются в стопку),
  заголовки «Мои задачи» / «Задачи мне», список строк;
- пусто → дружелюбная заглушка («нет залежавшихся задач»).

## Вне рамок (YAGNI)

- настраиваемый порог дней — не делаем, показываем просто ТОП-10 самых давних;
- changelog/история статусов — не трогаем, `jira_updated_at` достаточно;
- включение/выключение плитки — через существующий механизм enabled_widgets.

## Проверка

- миграция накатывается и откатывается (batch mode);
- адаптер: на сидовых данных два списка, правильная сортировка, лист-фильтр,
  завершённые отсечены, автор/исполнитель определяются по account_id;
- виджет рендерится, ссылки ведут в Jira, дни считаются от даты последнего касания.
