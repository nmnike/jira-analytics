# Дашборд · Помощь извне в виджете «Проекты квартала»

## Проблема

Виджет «Проекты квартала» считает суммарный факт по эпикам сценария **включая ворклоги людей вне команды**. На примере Q2 2026 «Команда 1С (ERP - Товарный учет)»: 276ч общего факта = 204ч команды + 72ч помощников извне. Из-за этого:

1. Метрика «Средняя загрузка» (276/1020 = 27%) завышена — командные ресурсы кажутся занятее, чем по факту (правильно 204/1020 = 20%).
2. Не видно, кто помогал и где — теряется информация о вкладе соседних команд.
3. Расхождение с виджетом «Ворклоги по категориям» (там 254ч для категории «Квартальные») — пользователь не понимает почему числа разные.

## Цели

- Показать чужие часы как **отдельную сущность** (надбавка, не вычитание).
- Командная загрузка % считается только от команды: `team_fact / plan`.
- Видно где именно была помощь и кто помогал.
- Текущая визуальная структура виджета сохраняется (бублик, легенда статусов, таблица проектов, KPI-стопка справа).

## Решение — Вариант 2

### Изменения в шапке (KPI-стопка справа)

KPI «Всего фактом» теперь показывает только командный факт. Добавляется новая карточка «Помощь извне».

| Карточка | Старое значение | Новое значение |
|---|---|---|
| Всего фактом | 276 ч из 1020 план | **204 ч** из 1020 план |
| Средняя загрузка | 27% | **20%** (факт команды / план) |
| **Помощь извне** (новая) | — | **+73 ч**, 6 чел, 5 проектов |
| Молчат > 14 дней | 2 | 2 (не меняется) |
| Закроются в срок | 5 | 5 (не меняется) |

Карточка «Помощь извне» визуально выделена мятно-зелёным (#84cc16 текст, мягкая зелёная подложка) — поддержка, не тревога.

### Изменения в таблице проектов

Между колонками «Задачи» и «Прогресс» добавляется узкая колонка **«Помощь»**:

- Стек аватаров помощников (максимум 3, дальше +N).
- Аватар — кружок 18×18 с инициалами, мятно-зелёный градиент.
- Справа от стека — суммарные часы помощи по проекту: «+8ч» мятно-зелёным.
- Если помощи на проекте не было — ячейка пустая (не отображается «—»).
- Hover на аватаре показывает имя помощника и его часы по проекту.

### Прогресс-бар и колонка «Факт/План»

- Прогресс-бар: длина рассчитывается как `team_fact / plan` (без чужих).
- «Факт/План» в колонке: показывает только командный факт (например, 14 / 180), без учёта чужих.

### Бублик и легенда статусов

Без изменений. Статусы проектов считаются по `Issue.status_category` — это не зависит от того, кто работал.

## Расчёты (backend)

В `analytics_service.get_dashboard_projects`:

1. После сбора всех issue_id сценария (`all_wl_ids`), определить множество ID сотрудников команды: `team_employee_ids = SELECT employee_id FROM employee_teams WHERE team IN (teams)`.

2. Группировать ворклоги дважды (два SQL-запроса):
   - **Командные ворклоги**: те же фильтры что сейчас + `Worklog.employee_id IN team_employee_ids`. → `team_fact_secs_by_issue`.
   - **Чужие ворклоги**: те же фильтры + `Worklog.employee_id NOT IN team_employee_ids`. → `alien_fact_secs_by_issue`.

3. Per-epic функции:
   - `epic_team_fact_hours(epic_id)` — сумма team_fact по эпику + детям.
   - `epic_alien_fact_hours(epic_id)` — то же для чужих.
   - `epic_alien_employees(epic_id)` — словарь `employee_id → секунды` чужих помощников, top-3 по часам.

4. Тренд/прогноз/forecast — пересчитываются от **команды**, не от total.

5. Топ-3 assignees строки (existing logic) — оставляем как сейчас (показывает всех, включая чужих, это другая колонка про «кто работал в принципе»). **Уточнить с пользователем при ревью.**

6. Если `teams` пуст (не выбрана команда), `team_employee_ids = {all employee_ids}` → все считаются «своими», alien = 0. Виджет в этом случае работает как раньше.

## API изменения

### Схема `ProjectItem`

Добавить:
```python
team_fact_hours: float          # часы команды по эпику
alien_fact_hours: float          # часы помощников извне
alien_helpers: list[ProjectAssignee]  # топ-3 помощника (инициалы + цвет)
alien_helper_count: int          # сколько всего помощников
```

Существующее поле `fact_hours` остаётся как `team_fact_hours + alien_fact_hours` для обратной совместимости — но фронт переключается на `team_fact_hours` для прогресс-бара и «Факт/План».

### Схема `DashboardProjectsResponse`

Добавить:
```python
total_alien_hours: float        # сумма помощи по всем проектам
alien_helper_count: int          # уникальные помощники
alien_projects_count: int        # сколько проектов получили помощь
```

`total_fact_hours` остаётся как `total_team_hours + total_alien_hours`; `avg_load_pct` пересчитывается от команды: `total_team_hours / total_plan_hours * 100`.

**Обратная совместимость:** старые поля остаются, новые добавляются. Если фронт не обновится — продолжит работать со старыми полями (показывая total, как сейчас).

## Фронт

`DashboardProjectsWidget.tsx`:

1. Колонка «Помощь» между «Задачи» и «Прогресс» — узкая (~80px). Содержит компонент `<AlienHelpersCell helpers={item.alien_helpers} count={item.alien_helper_count} hours={item.alien_fact_hours} />`.
2. KPI-карточка «Помощь извне» с `total_alien_hours`, `alien_helper_count`, `alien_projects_count` — добавляется в правую стопку, между «Загрузка» и «Молчат».
3. Прогресс-бар и «Факт/План» в строке переключаются на `team_fact_hours`.
4. Карточка «Всего фактом» в шапке → `total_team_hours` (новое поле `total_fact_hours_team` в схеме).
5. «Средняя загрузка» → `avg_load_pct` (пересчитан backend-ом).

`AlienHelpersCell` — новый компонент. Аватарный стек, мятно-зелёный градиент. Tooltip на каждом аватаре: «Иванов · 5ч».

## Out of scope

- Виджет «Ворклоги по категориям» — отдельная логика (категория задачи), пока не трогаем.

- Конфигурируемый порог «top-N помощников» — пока хардкод 3.
